## Задание 5. Бинарные файлы.
### Текст задачи
Информация о багаже пассажира описывается массивом, где каждый элемент содержит название
единицы багажа (чемодан, сумка, коробка и т.д.) и ее массу. Дан файл, содержащий сведения о
багаже нескольких пассажиров. Выяснить, имеется ли пассажир, багаж которого состоит из
одной единицы багажа массой менее m кг.
### Алгорит решения
1. Реализован статический метод заполнения XML-файла FillXmlFile, который формирует структуру данных в виде списка списков, где каждый внутренний список представляет багаж одного пассажира. Данные задаются заранее: для каждого пассажира создаётся список объектов типа Luggage, содержащих название багажа и его массу.
2. После формирования структуры создаётся объект XmlSerializer, с помощью которого данные сериализуются и записываются в файл. По завершении записи файл закрывается.
3. Реализован метод FindPassenger, который выполняет поиск пассажира по заданному условию. Сначала файл открывается для чтения с помощью FileStream, затем с использованием XmlSerializer выполняется десериализация данных — преобразование XML-файла обратно в объектную структуру.
4. После этого с помощью цикла перебираются все пассажиры, и для каждого проверяется условие: если у пассажира ровно одна единица багажа и её масса меньше заданного значения, метод возвращает true. Если такой пассажир не найден, возвращается false.

## Задание 6. List 
### Текст задачи
Даны упорядоченные списки L1 и L2. Вставить элементы списка L2 в список L1, не нарушая его
упорядоченности.
### Алгорит решения
1. Реализован статический метод ввода списка ReadList, который создаёт объект List<int> и заполняет его значениями, вводимыми пользователем с клавиатуры. Пользователь вводит количество элементов списка, после чего в цикле вводятся сами числа. После ввода список сортируется методом Sort, чтобы обеспечить его упорядоченность.
2. Реализован метод вставки элементов InsertSorted, который принимает два упорядоченных списка L1 и L2. Для каждого элемента списка L2 выполняется поиск позиции вставки в списке L1, чтобы сохранить его упорядоченность. Для этого используется цикл перебора элементов списка L1 и сравнение значений. После нахождения корректной позиции элемент вставляется с помощью метода Insert. Если подходящая позиция не найдена, элемент добавляется в конец списка.
3. Реализован метод вывода PrintList, который построчно выводит элементы списка с использованием цикла for.

## Задание 7. LinkedList 
### Текст задачи
Подсчитать количество элементов списка L, у которых равные «соседи».
### Алгорит решения
1. Реализован статический метод ввода списка ReadLinkedList, который создаёт объект типа LinkedList<int> и заполняет его значениями, вводимыми пользователем с клавиатуры. Пользователь вводит количество элементов списка, после чего в цикле по одному вводит элементы, которые добавляются в список с помощью метода AddLast.
2. Реализован метод обработки списка CountEqualNeighbors, который подсчитывает количество элементов, у которых равные «соседи». Для этого выполняется обход связного списка с использованием указателя на текущий узел (LinkedListNode<int>). Проверка начинается со второго элемента и заканчивается предпоследним, так как только у них есть два соседа. Для каждого элемента сравниваются значение предыдущего и следующего узла. Если они равны, счётчик увеличивается.
3. Реализован метод вывода PrintLinkedList, который построчно выводит элементы связного списка с помощью обхода через Next.
