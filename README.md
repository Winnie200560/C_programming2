## Задание 6. List 
### Текст задачи
Даны упорядоченные списки L1 и L2. Вставить элементы списка L2 в список L1, не нарушая его
упорядоченности.
### Алгорит решения
1. Реализован статический метод ввода списка ReadList, который создаёт объект List<int> и заполняет его значениями, вводимыми пользователем с клавиатуры. Пользователь вводит количество элементов списка, после чего в цикле вводятся сами числа. После ввода список сортируется методом Sort, чтобы обеспечить упорядоченность.
2. Реализован метод вставки элементов InsertSorted, который принимает два упорядоченных списка L1 и L2. Для каждого элемента списка L2 выполняется поиск позиции вставки в списке L1, чтобы сохранить его упорядоченность. Для этого используется цикл перебора элементов списка L1 и сравнение значений. После нахождения корректной позиции элемент вставляется с помощью метода Insert. Если подходящая позиция не найдена, элемент добавляется в конец списка.
3. Реализован метод вывода PrintList, который построчно выводит элементы списка с использованием цикла for.

## Задание 7. LinkedList 
### Текст задачи
Подсчитать количество элементов списка L, у которых равные «соседи».
### Алгорит решения
1. Реализован статический метод ввода списка ReadLinkedList, который создаёт объект типа LinkedList<int> и заполняет его значениями, вводимыми пользователем с клавиатуры. Пользователь вводит количество элементов списка, после чего в цикле по одному вводит элементы, которые добавляются в список с помощью метода AddLast.
2. Реализован метод обработки списка CountEqualNeighbors, который подсчитывает количество элементов, у которых равные «соседи». Для этого выполняется обход связного списка с использованием указателя на текущий узел (LinkedListNode<int>). Проверка начинается со второго элемента и заканчивается предпоследним, так как только у них есть два соседа. Для каждого элемента сравниваются значение предыдущего и следующего узла. Если они равны, счётчик увеличивается.
3. Реализован метод вывода PrintLinkedList, который построчно выводит элементы связного списка с помощью обхода через Next.

## Задание 8. HashSet
### Текст задачи
Задан некоторый набор блюд в кафе. Определить для каждого из блюд, какие из них заказывали
все посетители, какие — некоторые из посетителей, и какие не заказывал никто.
### Алгорит решения
1. Метод ReadVisit считывает количество посетителей n с проверкой через TryParse. Для каждого посетителя вводится количество блюд k, затем названия блюд. Блюда хранятся в HashSet<string>, чтобы исключить повторы. Результат — список List<HashSet<string>>.
2. Метод Dishes анализирует заказы посетителей и делит блюда на три группы:
some — все блюда, которые заказал хотя бы один посетитель (объединение)
all — блюда, которые заказали все посетители (пересечение)
none — блюда, которые не заказал никто (разность с исходным набором)
3. Для формирования some выполняется добавление всех блюд всех посетителей в одно множество.
4. Для формирования all изначально берётся полный набор блюд, затем из него удаляются те, которые отсутствуют у хотя бы одного посетителя.
5. Для формирования none из полного набора удаляются все блюда, которые встречаются хотя бы у одного посетителя.
6. Метод PrintSet выводит элементы множества на экран.

## Задание 9. HashSet
### Текст задачи
Файл содержит текст на русском языке. Напечатать в алфавитном порядке все согласные буквы, которые входят ровно в одно слово.
### Алгоритм решения 
1. Реализован статический метод PrintConsonants, который читает содержимое текстового файла по заданному пути с помощью File.ReadAllText. Затем текст разбивается на отдельные слова с помощью метода SplitWords, в котором учитываются пробелы, знаки препинания и переносы строк.
2. После этого создаётся словарь Dictionary<char, int>, который используется для подсчёта количества слов, в которых встречается каждая согласная буква. Для каждого слова создаётся множество HashSet<char>, чтобы учитывать каждую букву только один раз в пределах одного слова.
3. Внутри цикла по словам выполняется обход символов строки. Каждая буква проверяется методом IsConsonant, который определяет, является ли символ русской согласной. Если буква является согласной, она добавляется в HashSet, чтобы избежать повторов внутри одного слова.
4. После обработки слова выполняется обновление словаря: каждая уникальная согласная буква увеличивает своё значение в словаре на 1, что означает, в скольких различных словах она встретилась.
5. После завершения обработки всех слов выполняется обход словаря, и на экран выводятся только те буквы, значение которых равно 1, то есть те согласные, которые встречаются ровно в одном слове.
6. Дополнительно реализован метод IsConsonant, который проверяет принадлежность символа к строке русских согласных букв с помощью метода IndexOf.
7. Также реализован вспомогательный метод SplitWords, который разбивает текст на слова с использованием массива разделителей и удаляет пустые элементы с помощью StringSplitOptions.RemoveEmptyEntries.
