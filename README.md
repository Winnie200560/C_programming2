## Задание 1 - 5. Текстовые файлы
### Текст задачи
1. Найти разность максимального и минимального элементов.
2. Вычислить минимальный элемент.
3. Переписать в другой файл строки, начинающиеся с заданного символа.
4. Получить в новом файле все компоненты исходного файла, которые делятся на m и не делятся
на n.
5. Информация о багаже пассажира описывается массивом, где каждый элемент содержит название
единицы багажа (чемодан, сумка, коробка и т.д.) и ее массу. Дан файл, содержащий сведения о
багаже нескольких пассажиров. Выяснить, имеется ли пассажир, багаж которого состоит из
одной единицы багажа массой менее m кг.
### Алгорит решения
1. Реализован статический метод заполения файла, который создает объект SteamWriter для записи в файл, с помощью цикла генерирует случайное число в диапазоне от 0 до 100 на каждой итерации и записывает его в новую строку. После завершения цила файл закрывается.
Метод DifferentMaxMin, который находит разность между максимальным и минимальным значением файла. Открываем файла для чтения с помощью StreamReader, до тех пор пока файл не закончился считываем строку, преобразовываем ее в число и последовательно обновляем значения минимума и максимума. Возвращаем разность.
Метод вывода содержимого файла PrintFile, построчно читает файл и выводит его содержимое.
2. Реализован метод заполения файла с несколькими числами в строку через пробел. Метод поиска минимума: читаем файл построчно, каждую строку разбиваем на числа (с помощью Split), преобразуем строки в число, сравниваем и обновляем минимум. Возвращаем найденный минимум.
3. Реализован метод CopyLine, которые копирует строки, начинающиеся с заданного символа. Открываем исходный файл для чтения и новый для записи. Читаем файл построчно и для каждой строки проверяем, чтобы она не была пустой и на совпадение с заданным символом. Если совпадает - записываем в новый файл.
4. Реализован статический метод заполнения бинарного файла: открываеи бинарный файл для записи, генерируем случайные числа и каждое число записывается в файл в бинарном виде. Метод фильтрации данных: открываем исходный файл для чтения и новый для записи, пока не конец файла - считываем число и проверяем условие (num % m == 0 && num % n != 0), если оно выполняется записываем в новый файл.
5. Реализована структура Luggage, содержащая название единицы багажа (строка) и её массу (вещественное число). Данная структура используется для хранения информации об элементах багажа одного пассажира. Реализован статический метод заполнения файла FillXmlFile, который формирует список пассажиров, где каждый пассажир представлен списком объектов типа Luggage. С помощью генератора случайных чисел создаётся количество пассажиров и количество единиц багажа у каждого пассажира. Для каждого элемента багажа случайным образом выбирается его название из заданного набора строк и масса в диапазоне от 1 до 50. После формирования всех данных используется XmlSerializer для сериализации списка в XML-файл. После завершения записи файл закрывается. Реализован метод поиска FindPassenger, который определяет наличие пассажира, багаж которого состоит ровно из одной единицы. Для этого файл открывается и десериализуется аналогично методу вывода. Далее выполняется обход всех пассажиров: если у пассажира количество предметов багажа равно 1 и масса единственной вещи меньше заданного значения m, метод возвращает true, иначе false. Реализован метод PrintXmlFile для отображения содержимого файла в читаемом виде, который построчно выводит всех пассажиров и их багаж после десериализации данных из XML-файла.

## Задание 6 - 10. 
### Текст задачи
6. Даны упорядоченные списки L1 и L2. Вставить элементы списка L2 в список L1, не нарушая его
упорядоченности.
7. Подсчитать количество элементов списка L, у которых равные «соседи».
8. Задан некоторый набор блюд в кафе. Определить для каждого из блюд, какие из них заказывали
все посетители, какие — некоторые из посетителей, и какие не заказывал никто.
9. Файл содержит текст на русском языке. Напечатать в алфавитном порядке все согласные буквы,
которые входят ровно в одно слово.
10. На вход программы подаются сведения о результатах соревнований по школьному многоборью.
Многоборье состоит из соревнований по четырем видам спорта, участие в каждом из которых
оценивается баллами от 0 до 10 (0 баллов получает ученик, не принимавший участия в
соревнованиях по данному виду спорта). Победители определяются по наибольшей сумме
набранных баллов. Известно, что общее количество участников соревнований не превосходит 100.
В первой строке вводится количество учеников, принимавших участие в соревнованиях, N. Далее
следуют N строк, имеющих следующий формат:
<Фамилия><Имя><Баллы>
Здесь <Фамилия>– строка, состоящая не более чем из 20 символов;<Имя>– строка, состоящая не
более чем из 15 символов;<Баллы> - строка, содержащая четыре целых числа, разделенных
пробелом, соответствующих баллам, полученным на соревнованиях по каждому из четырех
видов спорта. При этом <Фамилия> и <Имя>, <Имя> и <Баллы> разделены одним пробелом.
Примеры входных строк:
Иванова Мария 5 8 6 3
Петров Сергей 9 9 5 7
Напишите программу, которая будет выводить на экран фамилии и имена трех лучших
участников многоборья. Если среди остальных участников есть ученики, набравшие то же
количество баллов, что и один из трех лучших, то их фамилии и имена также следует вывести.
При этом имена и фамилии можно выводить в произвольном порядке.
### Алгорит решения
6. Реализован статический метод ввода списка ReadList, который создаёт объект List<int> и заполняет его значениями, вводимыми пользователем с клавиатуры. Пользователь вводит количество элементов списка, после чего в цикле вводятся сами числа. После ввода список сортируется методом Sort, чтобы обеспечить его упорядоченность. Реализован метод вставки элементов InsertSorted, который принимает два упорядоченных списка L1 и L2. Для каждого элемента списка L2 выполняется поиск позиции вставки в списке L1, чтобы сохранить его упорядоченность. Для этого используется цикл перебора элементов списка L1 и сравнение значений. После нахождения корректной позиции элемент вставляется с помощью метода Insert. Если подходящая позиция не найдена, элемент добавляется в конец списка. Реализован метод вывода PrintList, который построчно выводит элементы списка с использованием цикла for.
